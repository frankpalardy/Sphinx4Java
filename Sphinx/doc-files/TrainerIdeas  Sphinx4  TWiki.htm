In this page, we flesh out some of the requirements presented in TrainerRequirements. Mostly from Bhiksha's slides presented during the face-to-face meeting.

  • Must train any kind (refers to tying) of model
    • Separate the computation of a posteriori probabilities of states from the computation of state density parameters
    • Data vectors now become (data-vector, a posteriori probability) pairs
    • Share the buffers for all tied parameters according to the tying
  • Variable number of states per model
    • Straight forward even using current data structures
  • Mixtures with different numbers of Gaussians
    • Straight forward
  • Different kinds of densities at different states
    • Compute a posteriori probabilities of states
    • Data vectors now become (data-vector, a posteriori probability) pairs
    • Compute state density for any state from these pairs, by treating vectors as weighted data
      • Similar to the manner in which Gaussian densities are computed in the Sphinx-3
      • Not clear if sufficient-statistics based accumulators can be computed for all densities
      • Should be possible for exponential density families
  • Any kind of context
    • Define utterance HMM with models of largest desired context
    • Learn models for largest context from the a posteriori probabilities
    • For smaller contexts, integrate larger contexts by averaging out contexts
      • Done simply by sharing buffers
      • May also be obtained by multiple passes over the training data
  • Multiple models simultaneously
    • Define extended HMM with the multiple models
    • Baum-Welch computes a posteriori probabilities for all states of the HMM
      • When all models are on the same feature, these can be used directly for state density and transition matrix estimation
  • Simultaneous training of multiple feature streams or models
    • Construct sentence HMM using multiple feature streams/models
    • Use a posteriori probabilities from Baum Welch to update parameters

-- EvandroGouvea - 21 Jan 2003
